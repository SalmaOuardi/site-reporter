# Introduction

Site Reporter MVP is a voice-driven construction-report assistant. It records French chantier memos in the browser, sends them through Azure GPT‑4o-mini-transcribe for speech-to-text, uses a Mistral LLM to infer the right template and fields, and returns an editable report through a Streamlit dashboard backed by FastAPI.

# Getting Started

## Installation Process
1. Install Python 3.11+ and [uv](https://github.com/astral-sh/uv).
2. Clone this repository:  
   ```bash
   git clone <your-repo-url>
   cd site-reporter
   ```

## Software Dependencies
- Backend: FastAPI, Uvicorn, Pydantic, OpenAI SDK (Azure), python-dotenv.
- Frontend: Streamlit, Requests, pandas, python-dotenv.
- uv handles virtual environments per project automatically.

## Environment Variables
1. Backend  
   ```bash
   cd backend
   cp .env.example .env  # fill Azure OpenAI keys + deployment info
   ```
2. Frontend  
   ```bash
   cd ../frontend
   echo "BACKEND_URL=http://localhost:8000" > .env
   ```

## Latest Releases
This MVP tracks main; tag releases when promoting to staging. (No published packages yet.)

## API References
- HTTP base: `http://localhost:8000/api`
- Endpoints:
  - `POST /transcribe`
  - `POST /report/template`
  - `POST /report/generate`
  - `POST /pipeline/auto`

Documentation is inline via FastAPI’s autogenerated Swagger UI (`/docs`).

# Build and Test

```bash
# Backend dependencies
cd backend
uv sync

# Frontend dependencies
cd ../frontend
uv sync

# Run backend (terminal 1)
cd backend
uv run uvicorn app.main:app --reload

# Run frontend (terminal 2)
cd frontend
uv run streamlit run app.py

# Execute integration tests
cd backend
uv run pytest ../tests
```

# Contribute

1. Fork the repository or create a feature branch.
2. Use `uv run` within `backend/` or `frontend/` so dependencies stay isolated.
3. Add or update tests under `tests/` when changing backend logic.
4. Submit a pull request describing the change, manual test steps, and any Azure resources required.
